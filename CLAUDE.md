# Claude Code プロジェクト設定

## プロジェクト概要

このプロジェクトは、Claude Codeのカスタムコマンド集です。日常的によく使うプロンプトをコマンドとして保存し、再利用可能にすることで作業効率を向上させます。

## 重要なルール

### 言語設定
- **日本語を使用すること**：このプロジェクトでは、すべてのコミュニケーション、ドキュメント、コマンドの説明を日本語で行います

### カスタムコマンドの構造

各カスタムコマンドは以下の構成を取ります：

```
コマンド名/
├── input.md   # ユーザーが入力内容を記述
├── output.md  # Claude Codeが結果を出力
└── sample.md  # サンプル入力（test引数で使用）
```

この構成により：
- 複雑な入力内容を事前に準備できる
- 標準入力での誤入力を防げる
- 出力結果を容易にコピー＆ペーストできる
- 改行を含む結果も正確に扱える
- sample.md を参考にしながら input.md を作成できる

### 命名規則

カスタムコマンドは目的に応じて接頭辞を付けます：

- **gen-**: 生成系コマンド
  - 例: `gen-branch-naming`, `gen-sql`, `gen-code-naming`
  - 何かを新しく生成・作成する場合に使用

- **review-**: 添削・レビュー系コマンド
  - 例: `review-text`, `review-code`
  - 既存のものを改善・添削する場合に使用

### 引数の扱い

#### 基本方針
カスタムコマンドでは、**引数は使用せず、全ての設定を input.md に記述する**方式を推奨します。これにより：
- 設定内容が明確になる
- 複雑な入力も扱いやすい
- 再利用が容易になる

#### test 引数（全コマンド共通）
全てのカスタムコマンドは `test` 引数をサポートします：

```
/コマンド名 test
```

`test` が指定された場合、`コマンド名/sample.md` の内容を入力として実行します。これにより：
- コマンドの動作確認が簡単にできる
- sample.md を見ながら実際の使い方を学べる
- sample.md を参考に input.md を作成できる

実装例：
```markdown
$ARGUMENTS に `test` が指定されている場合は、コマンド名/sample.md の内容を入力として使用してください。
```

## カスタムコマンド作成のベストプラクティス

### 1. 明確な要件定義

各コマンドファイル（`.claude/commands/*.md`）には以下を含めます：

- 入力ファイルのパス（例: `gen-branch-naming/input.md`）
- 処理内容の詳細な説明
- 出力ファイルのパス（例: `gen-branch-naming/output.md`）
- 出力フォーマットの明示

### 2. サンプルデータの提供

**重要**: 各コマンドには必ずサンプルデータを含めてください。これにより精度が大幅に向上します。

```markdown
## サンプル

### 想定 input.md:
```（サンプル入力）```

### 想定 output.md:
```（期待される出力）```
```

### 3. 出力フォーマットの統一

出力は読みやすく、構造化された形式にします：

- セクション見出しには【】を使用
- 箇条書きで情報を整理
- 理由や説明を明記

### 4. ユーザビリティの考慮

- おすすめや最適解を最初に提示する
- 複数の候補を提供する（3-5個程度）
- 各選択肢の理由を説明する

## ディレクトリ構造

```
claude-spells/
├── CLAUDE.md                    # このファイル
├── README.md                    # プロジェクト説明
├── .gitignore                   # output.md を除外
├── .claude/
│   └── commands/                # カスタムコマンド定義
│       ├── gen-*.md
│       └── review-*.md
├── gen-*/                       # 生成系コマンド用ディレクトリ
│   ├── input.md                 # ユーザー入力
│   ├── output.md                # 実行結果（.gitignore対象）
│   └── sample.md                # サンプル入力
└── review-*/                    # 添削系コマンド用ディレクトリ
    ├── input.md                 # ユーザー入力
    ├── output.md                # 実行結果（.gitignore対象）
    └── sample.md                # サンプル入力
```

## 新しいコマンドの追加方法

1. 命名規則に従ってコマンド名を決定（`gen-*` または `review-*`）
2. `/gen-custom-command` を使用して自動生成、または手動で以下を作成：
   - `.claude/commands/コマンド名.md` - コマンド定義
   - `コマンド名/input.md` - ユーザー入力（空ファイル）
   - `コマンド名/output.md` - 実行結果（空ファイル）
   - `コマンド名/sample.md` - サンプル入力
3. README.mdのコマンド一覧に追加

## まとめ

このプロジェクトでは、日本語でのコミュニケーションを基本とし、再利用可能なカスタムコマンドを体系的に管理します。各コマンドは明確な命名規則に従い、サンプルデータを含めることで、高精度で一貫性のある結果を提供します。
